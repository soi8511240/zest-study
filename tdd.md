# TDD

## Test-Driven Development 테스트 주도개발
- 코드 작성 전 테스트 코드를 먼저 작성

Write the test -> Run Tests -> success -> Write the test <br>
Write the test -> Run Tests -> fail -> Write only enough Code -> Run Test => Refactoring
- 사용자 입장에서 코드를 작성
- 모든 요구 사항에 대해 점검
- 요구사항 분석 및 이해 -> 설계
- 시스템 전반적인 설계 향상
- 개발 집중력 향상
